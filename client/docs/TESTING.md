# Тестирование

В проекте добавлены автоматизированные тесты для клиентской игровой логики и backend-сервиса.

## Клиентские тесты

Клиентские тесты находятся в каталоге `client/app/src/test` и проверяют чистую игровую логику без запуска Android UI.

Покрываются следующие компоненты:

- `GameEngineTest` — проверка начального состояния игры, генерации объектов и мобов, паузы, возобновления, сброса партии и базовых инвариантов сущностей;
- `GameStateTest` — проверка значений по умолчанию для игрока и состояния игры, параметров типов мобов и допустимости игровых констант;
- `ScoreCalculatorTest` — проверка формулы расчета очков на клиенте;
- `AnimClipTest` — проверка выбора кадров анимации, циклического проигрывания и параметров анимационных клипов.

Также в проекте сохранены базовые шаблонные тесты Android-проекта:

- `ExampleUnitTest` — простой unit-тест;
- `ExampleInstrumentedTest` — проверка package name в instrumented-среде.

Запуск клиентских unit-тестов:

```bash
cd client
./gradlew test
```

Instrumented-тесты запускаются отдельно на устройстве или эмуляторе:

```bash
cd client
./gradlew connectedAndroidTest
```

## Серверные тесты

Серверные тесты находятся в каталоге `server/src/test` и проверяют бизнес-логику backend API.

Покрываются следующие компоненты:

- `AuthControllerTest` — регистрация пользователя, запрет повторного username, вход с корректным паролем и отказ при неправильном пароле;
- `LeaderboardControllerTest` — отправка результата партии, проверка авторизации, валидация payload, ограничения на значения, публичный топ, сортировка, дедупликация пользователей и личная статистика игрока;
- `ScoreCalculatorTest` — проверка серверной формулы расчета очков.

Для тестового окружения используется конфигурация `server/src/test/resources/application-test.yml`.

Запуск серверных тестов:

```bash
cd server
./gradlew test
```

## Назначение тестов

Тесты фиксируют ключевые инварианты MVP:

- игра стартует в корректном состоянии;
- карта содержит игровые объекты и мобов;
- параметры игрока и мобов находятся в допустимых диапазонах;
- сброс партии восстанавливает состояние;
- счет считается одинаково на клиенте и сервере;
- backend корректно обрабатывает авторизацию и лидерборд;
- некорректные данные результата партии отклоняются сервером.
