## Процесс разработки
1.  Проектирование контрактов: Написание дата-классов GameState, Entity, Item до реализации систем.
2.  Изоляция модулей: Каждый разработчик работает в собственном пакете/модуле (напр. :feature:inventory, `:core:physics`).
3.  Entity-First подход: Логика пишется без привязки к Compose. Визуализация состояния происходит только после тестирования изменения данных в логах.

*   Базовая система сущностей (Player, Мобы) и их перемещение (Virtual Joystick -> Вектор скорости).

Колесова Софья: Entities Logic (AI и Боевая система)
*   Разработка стейт-машины для AI мобов и животных (Idle, Patrol, Chase, Attack, Flee).

Подмарев Александр: UI, Meta & Progression (Интерфейс и Мета-игра)
*   Разработка HUD (Шкала здоровья, голода, быстрый инвентарь, виртуальный джойстик).
